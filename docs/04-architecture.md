│                   ユーザー                        │

└──────┬────────────────────────────┬───────────────┘
       │                            │
       ▼                            ▼
┌──────────────┐           ┌──────────────────┐

│              │           └──────────────────┘
│  - ダッシュボード  │

│  - API Routes    │
└──────┬───────────┘
       │
       ▼
┌──────────────────┐
│  Supabase        │
│  (PostgreSQL)    │
│                  │

## 技術スタック

| レイヤー | 技術 | 理由 |
|----------|------|------|
| フロントエンド | Next.js 14 (App Router) + TypeScript | React ベース、SSR対応、PWA化容易 |
| UI | Tailwind CSS + カスタムコンポーネント | 高速開発、レスポンシブ対応 |
| バックエンド | Next.js API Routes | フロントと同一プロジェクト |
| データベース | Supabase (PostgreSQL) | 無料、REST API自動生成、リアルタイム対応 |
| 通知 | LINE Messaging API | ユーザーが最も見るアプリ |
| ホスティング | Vercel | Next.js最適、無料、自動デプロイ |
| 定期実行 | GitHub Actions (cron) | 既にリポジトリにある仕組み活用 |
| 認証 | 環境変数 (シングルユーザー) | 個人利用なのでシンプルに |

## 自動化フロー

### 毎日自動で起きること（ユーザー操作ゼロ）

1. **毎朝 8:00** - GitHub Actions が起動
2. 利息の日割り計算を実行
3. 今日の使える金額を計算
4. ゲーム状態を更新（デイリーXP付与）
5. LINE にモーニングレポート送信
6. ダッシュボードのデータを更新

### ユーザーがやること

| 頻度 | 作業 | 所要時間 |
|------|------|----------|
| 初回のみ | 借金情報の入力 | 10分 |
| 月1回 | 返済額の確認・更新（任意） | 2分 |
| 毎日 | LINE通知を見る | 30秒 |
| 気が向いた時 | ダッシュボードを開く | - |

## なぜこの構成がベストか

1. **怠惰ユーザー対応**: 初回設定後、手動操作ゼロで回る
2. **ゲーム要素フル実装**: ビジュアルUI でボスバトル・レベルアップを体験
3. **費用ゼロ**: 全て無料枠内
4. **iPhone対応**: PWAでホーム画面にアプリとして追加
5. **LINE通知**: 毎日自然に目に入る仕組み
6. **拡張性**: 後からMoney Forward連携等も追加可能
